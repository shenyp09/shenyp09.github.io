Github，Gitcafe以及Coding比较

By Y.P.Shen

发表于 1月 14 2015

今天突发奇想，想要再次尝试建立个博客，很自然的就需要寻找相应的代码托管平台来托管博客的代码。作为一个正常人，我第一个想到的当然是 Github ，不过鉴于GFW的突出贡献， Github 在国内的速度实在是不敢恭维，因此希望能够在找一个国内的代码托管平台来代替。

由于微博上关注了 Coding 的官微，所以经常看到与 Coding 相关的新闻，所以对其印象比较好。加之最近正在建设的 JUNA 的网站的代码也托管在 Coding 上，经过这段时间的使用，其速度还是给我留下了深刻的印象。因此，似乎把博客托管到 Coding 上是个不错的选择。

博客使用的是 Hexo 来搭建的。 Wordpress 什么的早就不想了，因为实在是太臃肿了，总是被技术、设计等细枝末节的问题吸引（好吧，我就是个找不到重点的人）而导致无法把精力投入到写作当中。而 Hexo 带给我最大的惊喜就是我只要专注于写作就好了，其他的根本无需纠结（我也懒得纠结， Jacman 的主题相当的不错）。额，好像又没找到重点。

说回 Coding ，由于JUNA的网站并没有托管在 Coding 上，因此之前一直没有了解在 Coding 中怎么托管静态网站，至于 Hexo 怎么直接部署在 Coding 上更是完全没有头绪。上网搜了一下，发现倒是有办法托管上去，但是很快就遇到了一个大坑。 Coding 上部署静态网站实际上是PaaS，而我每次更新博客居然在 Coding 上重新部署服务器才行。这真是个痛苦的过程，博客不像其他静态网站，修改频率相当高，每次都部署实在是太麻烦了。我瞬间就绝了在 Coding 上部署博客的念想。

剩下的似乎只有 GitCafe 可以尝试了。其实在很久以前我曾经使用过 GitCafe 的，在它上面部署静态网页跟 Github 很类似，体验也还不错。但是最大的坑就是它的速度根本不像是个国内的网站（或者根本就不是？）！我抱着过了这么久了应该会有所改进的期望，重新登入了 GitCafe （密码都快忘掉了），果不其然，还是一如既往的慢。呵呵。。。

最终我还是重新回到了 Github 的怀抱，虽然速度慢点儿，好在每次上传的代码量比较少，总算还能凑合着用。

PS：貌似 Coding 是有自动更新的功能的，但是我按照指导来配置发现根本无法成功。而且部署的速度时快时慢，有时候要十几分钟。年龄大了，懒得折腾了。

主题介绍

Jacman 是一款扁平化，有着响应式设计的 Hexo 主题。本站正式使用了 Jacman 主题。Jacman 基于 Pacman 主题修改而来，目前已有很多特性与原主题不同。你可以前往 本站 和 Demo 预览更多关于本主题的更多效果。如果你有任何问题或意见欢迎到 GitHub 发表 issue。

距离当初我把 Jacman 开源至今已有半年零一周了，在这半年里也一直保持着对 Jacman 的更新。看到很多人用这款主题，fork&star 数也越来越好看，我也更有动力继续完善 Jacman。Jacman 自然还有很多改进空间，首先是文档问题，有许多人通过博客、微博、QQ 询问我有关主题的问题，我也意识到之前写的帮助手册可读性太差。正好趁着这次 Jacman 大更新，写篇更详尽、readable 的手册。

Red Monk的语言排名：Java Script，Java保持领先；Go未进前20

JavaScript在基于Github
和Stack Overflow
的排名统计中，均排名第一。

在RedMonk的这个月编程语言指数中：JavaScript，Java和PHP排名前三。这个指数基于Github
的语言代码行数和Stack Overflow上的语言标签数量。

“在狭隘的比较中，Java总是在排名中超过JavaScript，但二者的差异总是如此之远，以至于并不显著”，RedMonk首席分析师Stephen
O’Grady这样说。“最显著的特点是该语言经常被描述为将死，有时又被吹捧地上升几位”

RedMonk的指数中 Python, C#, C++, Ruby, CSS, C, 和
Objective-C也进入前十。排名11~20的语言为：Perl 和 Shell (并列11), R,
Scala, Haskell, Matlab, Go 和 Visual Basic (并列17), Clojure 和 Groovy
(并列19)。

在O’Grady认为有几种语言的排名指数还没有什么考查价值，包括Go, R 和
Switft。“在我们最近的指数轨迹分析中，Go进入前20还需要6~12个月的时间”，他说：“在半年以内，Go就可以超越Visual
Basic，Clojure和Groovy，取代CoffeeScript，进入第17名”。

R语言连续四个季度排在第13位。“即使R语言保持在13的位置，它仍然是最流行的统计语言。”，
O’Grady说，“虽然它面临着Python这样通用替代语言的竞争”。

O’Grady认为Swift，这个Apple的新语言是一个“神奇”的例子，它从上次的22位上升到了68位。“从列表中非常靠下的位置，上升到了CoffeeScript的后面和Lua的前面”。根据现在的统计数据，在今年的第三季度，它很可能进入前20。

CoffeeScript在出现的两年后跌出了前20.“这种编绎成JavaScript的语言将自己定位为既有JavaScript的流行，又避免了它的怪癖，但支持率似乎在逐渐下降”
，不过第三季度就可以看出来它是临时性的下降还是持续性的。

RedMonk指数是另外一个比较蓄积语言流行度的排名。每两年发表一次。它与
PYPL的编程语言排名（Pypl Popularity Programming Language
index）稍有不同：Java第一,PHP第二，JavaScript第7，PyPL将Google的搜索热度作为主要指标。另外一个排名：Tiobe，主要调查各主要网站，它的排名中C语言第一，Java第六，PHP第6，且JavaScript第7。但是JavaScript成为Tiobe
2014的年度语言。

Switch to Yilia

我打赌，我在前端方面绝对是个颜控。

Jacman 主题功能非常完善，使用也很方便。但是感觉主题风格还是不是很合我的胃口。扁平化流行了相当一段时间了，我感觉我也被带沟里了。在网上随便一搜，很快就找到了一个中意的主题。

Github 上搜索 Hexo theme ，排名第一的是 Pacman ，其次就是 Yilia，一看 DEMO 瞬间就喜欢上这个主题了。扁平化非常到位，用色也比较收敛，很符合我的胃口。因此我果断地背叛了 Jacman，转投 Yilia 的怀抱了。不过虽然界面相当漂亮，但是不知道是因为作者喜好极简的缘故还是开发完成度不高的缘故，这个主题配置的丰富程度远远不能和 Jacman 相比。不过这个对我倒暂时不算什么问题，经过昨天的琢磨，现在大概知道该怎么修改这些主题了。暂时只是随手添加了百度统计，看以后需要决定要不要添加更多的功能。当然，如果作者以后自己加上这些功能就太好了。

前言

Markdown+Pandoc，可以把自己的写作内容，变成世界上已有的任何格式的文件，包括很炫的slide，html5。没有人（或者我没看到）总结过这些内容，导致我走了很多弯路才最终打通任督二脉，特此纪念。了解Markdwon以后，我的写作世界，只有它；看到Pandoc格式转换以后，对生成的slide和pdf羡慕的不行。那时，自己期望以后的写作是这样的：首先用Markdown把自己的想法写下来；其次，通过Pandoc，把写好的Markdown文件，转换成Slide或者PDF。如此而已。

Github，Gitcafe以及Coding比较

今天突发奇想，想要再次尝试建立个博客，很自然的就需要寻找相应的代码托管平台来托管博客的代码。作为一个正常人，我第一个想到的当然是 Github ，不过鉴于GFW的突出贡献， Github 在国内的速度实在是不敢恭维，因此希望能够在找一个国内的代码托管平台来代替。

由于微博上关注了 Coding 的官微，所以经常看到与 Coding 相关的新闻，所以对其印象比较好。加之最近正在建设的 JUNA 的网站的代码也托管在 Coding 上，经过这段时间的使用，其速度还是给我留下了深刻的印象。因此，似乎把博客托管到 Coding 上是个不错的选择。

博客使用的是 Hexo 来搭建的。 Wordpress 什么的早就不想了，因为实在是太臃肿了，总是被技术、设计等细枝末节的问题吸引（好吧，我就是个找不到重点的人）而导致无法把精力投入到写作当中。而 Hexo 带给我最大的惊喜就是我只要专注于写作就好了，其他的根本无需纠结（我也懒得纠结， Jacman 的主题相当的不错）。额，好像又没找到重点。

说回 Coding ，由于JUNA的网站并没有托管在 Coding 上，因此之前一直没有了解在 Coding 中怎么托管静态网站，至于 Hexo 怎么直接部署在 Coding 上更是完全没有头绪。上网搜了一下，发现倒是有办法托管上去，但是很快就遇到了一个大坑。 Coding 上部署静态网站实际上是PaaS，而我每次更新博客居然在 Coding 上重新部署服务器才行。这真是个痛苦的过程，博客不像其他静态网站，修改频率相当高，每次都部署实在是太麻烦了。我瞬间就绝了在 Coding 上部署博客的念想。

剩下的似乎只有 GitCafe 可以尝试了。其实在很久以前我曾经使用过 GitCafe 的，在它上面部署静态网页跟 Github 很类似，体验也还不错。但是最大的坑就是它的速度根本不像是个国内的网站（或者根本就不是？）！我抱着过了这么久了应该会有所改进的期望，重新登入了 GitCafe （密码都快忘掉了），果不其然，还是一如既往的慢。呵呵。。。

最终我还是重新回到了 Github 的怀抱，虽然速度慢点儿，好在每次上传的代码量比较少，总算还能凑合着用。

PS：貌似 Coding 是有自动更新的功能的，但是我按照指导来配置发现根本无法成功。而且部署的速度时快时慢，有时候要十几分钟。年龄大了，懒得折腾了。
